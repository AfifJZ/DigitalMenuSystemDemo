# Railway Deployment & Environment Configuration

## Critical: Update PUBLIC_BASE_URL on Railway

The QR code links will fail with 404 errors if `PUBLIC_BASE_URL` is not set to the correct working Railway domain.

### Steps to Fix:

1. **Go to Railway Dashboard**: https://railway.app
2. **Select your project**: Digital Menu System
3. **Select the service**: (the one you want QR codes to point to)
4. **Click on the service** → **Variables** tab
5. **Find or add `PUBLIC_BASE_URL`** and set it to:
   ```
   https://digitalmenusystem.up.railway.app
   ```
6. **Save** - Railway will automatically redeploy with the new variable

### Why This Matters:

- QR codes generated by the app use `PUBLIC_BASE_URL` to build links
- If `PUBLIC_BASE_URL` points to an inactive or wrong domain, customers get 404 errors
- The correct URL format: `https://your-service-name.up.railway.app` (no trailing slash)

## Verification

After updating `PUBLIC_BASE_URL`:

1. Go to the staff dashboard: `https://digitalmenusystem.up.railway.app/manage-login`
2. Navigate to **Manage** → **QR Codes** for a branch
3. Check that the generated links start with `https://digitalmenusystem.up.railway.app/customer?branch=...`
4. Scan a QR code or visit the link in a browser - it should load without 404

## Troubleshooting

**Still getting 404 after updating PUBLIC_BASE_URL?**

1. Clear your browser cache (Ctrl+Shift+Delete)
2. Wait 1-2 minutes for Railway to redeploy with the new env var
3. Check the server logs in Railway dashboard for any errors
4. Verify the environment variable was saved (refresh the page to confirm)
